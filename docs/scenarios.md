# Демо-сценарии MarketingSelector

Три сценария для проверки приложения MarketingSelector.

## Сценарий 1: Штатный (сбалансированный)

**Цель:** проверить работу приложения в стандартном режиме с равными весами.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 5.0 |
| Вес «CAC» | 5.0 |
| Вес «Релевантность» | 5.0 |
| Бюджет | 1 000 000 руб |
| Маржа | 25% |
| Конверсия | 3% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | 1. Telegram (0.44), 2. Поисковая реклама (0.28), 3. YouTube (0.28) |
| Платёжная матрица | 2×2, асимметричная (B != A.T), CAC влияет на выигрыш |
| Равновесие Нэша | Смешанное: (Telegram 11.6%, Поисковая реклама 88.4%) для обоих игроков |
| Ожидаемая прибыль | Игрок 1: 11 165 698 руб, Игрок 2: ~13 800 000 руб |
| Рекомендация | **Поисковая реклама** — 88.4% бюджета |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить ползунки весов на 5.0, остальные параметры — по умолчанию. Нажать «Рассчитать».

## Сценарий 2: Агрессивный охват

**Цель:** продемонстрировать расхождение между АИИ-ранжированием и игровым равновесием: YouTube лидирует по охвату, но проигрывает в конкурентной игре из-за высокого CAC.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 10.0 |
| Вес «CAC» | 0.0 |
| Вес «Релевантность» | 0.0 |
| Бюджет | 2 000 000 руб |
| Маржа | 30% |
| Конверсия | 5% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | 1. YouTube (0.47), 2. Поисковая реклама (0.33), 3. Telegram (0.20) |
| Платёжная матрица | 2×2, асимметричная, CAC влияет на прибыль |
| Равновесие Нэша | Смешанное: (Поисковая реклама 92.3%, YouTube 7.7%) для обоих игроков |
| Ожидаемая прибыль | Игрок 1: 21 019 231 руб, Игрок 2: 22 219 231 руб |
| Рекомендация | **Поисковая реклама** — 92.3% бюджета |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить: Охват=10, CAC=0, Релевантность=0, Бюджет=2 000 000, Маржа=30%, Конверсия=5%. Нажать «Рассчитать».

**Примечание:** YouTube побеждает в АИИ-ранжировании благодаря максимальному охвату (70%), но высокий CAC (5 000 руб) резко снижает чистую прибыль на клиента. В конкурентной игре Поисковая реклама с более скромным охватом (50%), но низким CAC (2 000 руб) оказывается выгоднее. Это ещё один пример расхождения между многокритериальной оценкой (АИИ) и игровым равновесием (теория игр).

## Сценарий 3: Эконом-режим (минимизация CAC)

**Цель:** проверить, что при максимальном весе CAC система выбирает Telegram с CAC=500.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 0.0 |
| Вес «CAC» | 10.0 |
| Вес «Релевантность» | 0.0 |
| Бюджет | 500 000 руб |
| Маржа | 15% |
| Конверсия | 2% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | 1. Telegram (0.74), 2. Поисковая реклама (0.19), 3. YouTube (0.07) |
| Платёжная матрица | 2×2, умеренные значения из-за малого бюджета |
| Равновесие Нэша | Смешанное: (Telegram 11.6%, Поисковая реклама 88.4%) для обоих игроков |
| Ожидаемая прибыль | Игрок 1: 4 366 279 руб, Игрок 2: 8 932 558 руб |
| Рекомендация | **Поисковая реклама** — 88.4% бюджета |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить: Охват=0, CAC=10, Релевантность=0, Бюджет=500 000, Маржа=15%, Конверсия=2%. Нажать «Рассчитать».

**Примечание:** Telegram выигрывает АИИ-ранжирование благодаря низкому CAC, но в экономической игре более высокий охват Поисковой рекламы компенсирует разницу в CAC. Это демонстрирует различие между многокритериальной оценкой (АИИ) и конкурентным равновесием (теория игр).
