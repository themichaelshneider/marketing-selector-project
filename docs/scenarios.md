# Демо-сценарии MarketingSelector

Три сценария для проверки приложения MarketingSelector.

## Сценарий 1: Штатный (сбалансированный)

**Цель:** проверить работу приложения в стандартном режиме с равными весами.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 5.0 |
| Вес «CAC» | 5.0 |
| Вес «Релевантность» | 5.0 |
| Бюджет | 1 000 000 руб |
| Маржа | 25% |
| Конверсия | 3% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | Telegram — 1-е место (низкий CAC, высокая релевантность) |
| Платёжная матрица | 3×3, значения в рублях |
| Равновесие Нэша | Вероятностное распределение бюджета между топ-3 каналами |
| Рекомендация | Доля бюджета > 30% на канал-лидер |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить ползунки весов на 5.0, остальные параметры — по умолчанию. Нажать «Рассчитать».

## Сценарий 2: Агрессивный охват

**Цель:** продемонстрировать, что при максимальном весе критерия «Охват» лидирует ТВ.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 10.0 |
| Вес «CAC» | 0.0 |
| Вес «Релевантность» | 0.0 |
| Бюджет | 2 000 000 руб |
| Маржа | 30% |
| Конверсия | 5% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | ТВ — 1-е место (максимальный охват 80%) |
| Платёжная матрица | Все значения могут быть отрицательными из-за высокого бюджета |
| Равновесие Нэша | Распределение с перекосом в сторону канала с наибольшим охватом |
| Рекомендация | ТВ получает наибольшую долю бюджета |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить: Охват=10, CAC=0, Релевантность=0, Бюджет=2 000 000, Маржа=30%, Конверсия=5%. Нажать «Рассчитать».

## Сценарий 3: Эконом-режим (минимизация CAC)

**Цель:** проверить, что при максимальном весе CAC (стоимость привлечения) и минимальном бюджете система выбирает Telegram с CAC=300.

**Параметры запуска:**

| Параметр | Значение |
|---|---|
| Вес «Охват» | 0.0 |
| Вес «CAC» | 10.0 |
| Вес «Релевантность» | 0.0 |
| Бюджет | 500 000 руб |
| Маржа | 15% |
| Конверсия | 2% |

**Ожидаемый результат:**

| Этап | Ожидание |
|---|---|
| АИИ-ранжирование | Telegram — 1-е место (CAC = 300, минимальный среди всех) |
| Платёжная матрица | Умеренные значения прибыли из-за малого бюджета |
| Равновесие Нэша | Значительная доля бюджета направляется на Telegram |
| Рекомендация | Telegram > 40% бюджета |

**Команда:**
```bash
streamlit run ui/app.py
```
Установить: Охват=0, CAC=10, Релевантность=0, Бюджет=500 000, Маржа=15%, Конверсия=2%. Нажать «Рассчитать».
